# Live Activity 开发按钮设计

## 目标

让开发者在不导入课表的情况下，从应用内手动创建 EveryClass 的演示 Live Activity，以便快速检查锁屏与灵动岛显示效果。

## 范围

在设置页的“实时通知”分区下增加一个仅在 Flutter `kDebugMode` 中构建的开发入口。点击入口后复用现有 `LiveNotification.runDemo()`，创建五分钟的固定演示课程。发布和 Profile 构建中不渲染此入口，也不自动创建演示通知。

## 交互与错误处理

按钮使用现有 `ListTile` 样式，标题为“开发：预览实时活动”，副标题说明它会显示五分钟演示课程。点击后等待原生调用完成：返回 `true` 时显示“已启动演示实时活动”，返回 `false` 时显示“当前设备不支持或未启用实时活动”。

入口不改变实时通知开关、倒计时设置、课表数据或持久化状态。重复点击沿用原生层的单 Activity 复用语义，更新同一个演示活动。

## 验证

为 SettingsScreen 增加 widget 测试，验证 Debug 模式可见入口且点击后调用演示通知并显示成功/失败反馈。运行 `flutter analyze` 与完整 `flutter test`；iOS 模拟器手工点击该入口，确认锁屏和灵动岛出现原生活动。
